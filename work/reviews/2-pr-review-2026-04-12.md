### Important

**[1] EMPTY-HANDED exits lose destination — inconsistent with DEFLATE pattern**
File: `resources/1dungeon.edn` — `timber-room :west`, `lower-shaft :out`, `lower-shaft :east`

ZIL uses `IF EMPTY-HANDED ELSE "..."` on these exits, with destinations `:lower-shaft` and `:timber-room`. EDN encodes them as bare blocking strings, discarding the destination. The DEFLATE pattern (used on `white-cliffs-north`/`south`) preserves the destination: `{:to :room :if :deflate-flag :else "msg"}`. EMPTY-HANDED should follow the same pattern with `:empty-handed-flag`.

Current (drops destination):
```clojure
:west "You cannot fit through this passage with that load."
```
Should be:
```clojure
:west {:to :lower-shaft :if :empty-handed :else "You cannot fit through this passage with that load."}
```
Same fix for `lower-shaft :out`/`:east`. Without this, `:lower-shaft` and `:machine-room` are unreachable by traversal (only reachable via direct `reset! z/here`).

**[2] Missing test: :if-open invisible fallthrough behavior**
File: `tests/integration/zork.clojure/underground_rooms_test.clj`

The v-walk `:if-open` patch has a critical corner: when the gating object has `:invisible` set, it ignores `:else` and prints "You can't go that way." instead. This is ZIL-faithful behavior (keeps trap-door working before rug puzzle), but there is no test locking it in. A regression here would make `:else` strings show up for invisible objects — breaking the pre-rug trap-door behavior.

Add a test: grate still invisible + `go up` from grating-room → "You can't go that way." (not "The grating is closed.").
