### Important

**[IMPORTANT] describe-objects does not suppress :invisible objects**
File: `src/zork.clojure/zork1/actions.clj:268`

The `describe-objects` function checks `:ndescbit` to suppress auto-description, but not
`:invisible`. In ZIL, the INVISIBLE flag also prevents objects from being auto-described
when looking at a room. This PR introduces four objects with `:invisible` but no `:ndescbit`
that will incorrectly appear in room descriptions:

- `:thief` (`:round-room`) ā€” has `:ldesc`; would print description of thief before he should
  be visible
- `:trunk` (`:reservoir`) ā€” has `:fdesc`; would print "Lying half buried in the mud..."
  before it should be visible
- `:pot-of-gold` (`:end-of-rainbow`) ā€” has `:fdesc`; would print "At the end of the
  rainbow is a pot of gold." on first look
- `:scarab` (`:sandy-cave`) ā€” no ldesc/fdesc; would print "There is a beautiful jeweled
  scarab here." on first look

Recommendation: Add `:invisible` to the suppression check in `describe-objects`:
```clojure
(when-not (or (flag? obj :ndescbit) (flag? obj :invisible))
```

### Suggestions

**[SUGGESTION] :slide at :local-globals has no :ndescbit ā€” may unexpectedly appear**
File: `resources/1dungeon.edn` (:slide entry)

ZIL has `(FLAGS CLIMBBIT)` for SLIDE with no NDESCBIT. This matches the ZIL source faithfully.
However, if a room lists `:slide` in its `:globals`, players would see "There is a chute here."
when looking (no fdesc or ldesc). Worth verifying against frotz once room globals lookup
is implemented.

**[SUGGESTION] :global-water has no :ndescbit ā€” same concern as :slide**
File: `resources/1dungeon.edn` (:global-water entry)

No NDESCBIT in ZIL source (only DRINKBIT). If shown in a room, would print "There is a
water here." Verify once globals lookup is wired.
